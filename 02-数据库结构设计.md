# AI Work OS - 数据库结构设计

## 1. 数据库选型

### 1.1 PostgreSQL (主数据库)
- **用途**: 存储结构化数据
- **版本**: PostgreSQL 15+
- **扩展**: UUID-OSSP, pgcrypto, pg_trgm

### 1.2 Vector Database (向量数据库)
- **选项**: Pinecone / Weaviate / Qdrant
- **用途**: 存储Agent知识库、文档嵌入

### 1.3 Redis
- **用途**: 缓存、会话管理、任务队列
- **版本**: Redis 7+

## 2. PostgreSQL 表结构设计

### 2.1 用户与认证表

#### `users` - 用户表
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(255),
    avatar_url TEXT,
    role VARCHAR(50) DEFAULT 'user', -- 'user', 'admin', 'enterprise'
    is_active BOOLEAN DEFAULT true,
    is_verified BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP WITH TIME ZONE,
    metadata JSONB DEFAULT '{}'::jsonb
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_created_at ON users(created_at);
```

#### `api_keys` - API密钥表
```sql
CREATE TABLE api_keys (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    key_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    prefix VARCHAR(20) NOT NULL, -- 用于显示的前缀
    scopes JSONB DEFAULT '[]'::jsonb, -- 权限范围
    is_active BOOLEAN DEFAULT true,
    last_used_at TIMESTAMP WITH TIME ZONE,
    expires_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_api_keys_user_id ON api_keys(user_id);
CREATE INDEX idx_api_keys_prefix ON api_keys(prefix);
```

### 2.2 项目管理表

#### `projects` - 项目表
```sql
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    task_type VARCHAR(50) NOT NULL, -- 'research', 'product_design', 'software_development', 'business_analysis', 'startup_planning'
    status VARCHAR(50) DEFAULT 'draft', -- 'draft', 'planning', 'executing', 'reviewing', 'completed', 'failed', 'cancelled'
    priority INTEGER DEFAULT 0, -- 0-10
    
    -- 目标信息
    goal_raw TEXT NOT NULL, -- 用户原始输入
    goal_parsed JSONB, -- 解析后的结构化目标
    
    -- 时间信息
    started_at TIMESTAMP WITH TIME ZONE,
    completed_at TIMESTAMP WITH TIME ZONE,
    estimated_duration INTEGER, -- 预估时长(分钟)
    actual_duration INTEGER, -- 实际时长(分钟)
    
    -- 统计信息
    total_tasks INTEGER DEFAULT 0,
    completed_tasks INTEGER DEFAULT 0,
    total_messages INTEGER DEFAULT 0,
    total_decisions INTEGER DEFAULT 0,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    metadata JSONB DEFAULT '{}'::jsonb
);

CREATE INDEX idx_projects_user_id ON projects(user_id);
CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_task_type ON projects(task_type);
CREATE INDEX idx_projects_created_at ON projects(created_at DESC);
```

### 2.3 Agent相关表

#### `agents` - Agent实例表
```sql
CREATE TABLE agents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    
    -- Agent基本信息
    role VARCHAR(100) NOT NULL, -- 'CoordinatorAgent', 'PlannerAgent', 'ArchitectAgent', etc.
    name VARCHAR(255) NOT NULL,
    description TEXT,
    
    -- Agent配置
    agent_type VARCHAR(50) NOT NULL, -- 'coordinator', 'planner', 'specialist'
    capabilities JSONB DEFAULT '[]'::jsonb, -- ['code_execution', 'web_search', 'file_management']
    tools JSONB DEFAULT '[]'::jsonb, -- 可用工具列表
    
    -- LLM配置
    llm_provider VARCHAR(50), -- 'openai', 'anthropic', 'local'
    llm_model VARCHAR(100),
    llm_config JSONB DEFAULT '{}'::jsonb, -- temperature, max_tokens, etc.
    
    -- 权限配置
    authority_level INTEGER DEFAULT 1, -- 1-5, 决策权限级别
    can_approve BOOLEAN DEFAULT false,
    can_delegate BOOLEAN DEFAULT false,
    
    -- 状态信息
    status VARCHAR(50) DEFAULT 'idle', -- 'idle', 'working', 'waiting', 'completed', 'error'
    current_task_id UUID,
    
    -- 统计信息
    tasks_assigned INTEGER DEFAULT 0,
    tasks_completed INTEGER DEFAULT 0,
    messages_sent INTEGER DEFAULT 0,
    decisions_made INTEGER DEFAULT 0,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    metadata JSONB DEFAULT '{}'::jsonb
);

CREATE INDEX idx_agents_project_id ON agents(project_id);
CREATE INDEX idx_agents_role ON agents(role);
CREATE INDEX idx_agents_status ON agents(status);
```

### 2.4 任务管理表

#### `tasks` - 任务表
```sql
CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    
    -- 任务基本信息
    title VARCHAR(500) NOT NULL,
    description TEXT,
    task_type VARCHAR(100) NOT NULL, -- 'requirement_analysis', 'architecture_design', 'coding', 'testing', etc.
    
    -- 责任人
    owner_agent_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    reviewer_agent_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    approver_agent_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    
    -- 任务状态
    status VARCHAR(50) DEFAULT 'pending', -- 'pending', 'in_progress', 'blocked', 'review', 'approved', 'completed', 'failed'
    priority INTEGER DEFAULT 0, -- 0-10
    
    -- 依赖关系
    depends_on JSONB DEFAULT '[]'::jsonb, -- [task_id1, task_id2, ...]
    blocks JSONB DEFAULT '[]'::jsonb, -- [task_id1, task_id2, ...]
    
    -- 时间信息
    estimated_duration INTEGER, -- 预估时长(分钟)
    actual_duration INTEGER, -- 实际时长(分钟)
    started_at TIMESTAMP WITH TIME ZONE,
    completed_at TIMESTAMP WITH TIME ZONE,
    deadline TIMESTAMP WITH TIME ZONE,
    
    -- 输入输出
    input_data JSONB DEFAULT '{}'::jsonb,
    output_data JSONB DEFAULT '{}'::jsonb,
    
    -- 执行信息
    execution_logs JSONB DEFAULT '[]'::jsonb,
    error_message TEXT,
    retry_count INTEGER DEFAULT 0,
    max_retries INTEGER DEFAULT 3,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    metadata JSONB DEFAULT '{}'::jsonb
);

CREATE INDEX idx_tasks_project_id ON tasks(project_id);
CREATE INDEX idx_tasks_owner_agent_id ON tasks(owner_agent_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_created_at ON tasks(created_at DESC);
```

#### `task_dependencies` - 任务依赖关系表

```sql
CREATE TABLE task_dependencies (

    dependency_type VARCHAR(50) DEFAULT 'finish_to_start', -- 'finish_to_start', 'start_to_start', 'finish_to_finish'
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(predecessor_task_id, successor_task_id)

CREATE INDEX idx_task_dependencies_project_id ON task_dependencies(project_id);
CREATE INDEX idx_task_dependencies_predecessor ON task_dependencies(predecessor_task_id);

```

### 2.5 沟通与消息表

```sql
    -- 消息元数据

    -- 附加数据

    is_important BOOLEAN DEFAULT false,

    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,

CREATE INDEX idx_messages_task_id ON messages(task_id);
CREATE INDEX idx_messages_sender_agent_id ON messages(sender_agent_id);
CREATE INDEX idx_messages_receiver_agent_id ON messages(receiver_agent_id);
CREATE INDEX idx_messages_message_type ON messages(message_type);
```

### 2.6 决策记录表

#### `decisions` - 决策表
```sql
CREATE TABLE decisions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
    
    -- 决策信息
    decision_type VARCHAR(100) NOT NULL, -- 'technical', 'architectural', 'process', 'resource_allocation', etc.
    title VARCHAR(500) NOT NULL,
    description TEXT NOT NULL,
    
    -- 决策者
    made_by_agent_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    approved_by_agent_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    
    -- 决策内容
    options_considered JSONB DEFAULT '[]'::jsonb, -- 考虑过的选项
    chosen_option JSONB NOT NULL, -- 选择的方案
    rationale TEXT NOT NULL, -- 决策理由
    
    -- 影响分析
    impact_scope VARCHAR(50), -- 'task', 'project', 'system'
    affected_tasks JSONB DEFAULT '[]'::jsonb,
    affected_agents JSONB DEFAULT '[]'::jsonb,
    
    -- 状态
    status VARCHAR(50) DEFAULT 'proposed', -- 'proposed', 'approved', 'rejected', 'implemented', 'reverted'
    
    -- 时间信息
    proposed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    approved_at TIMESTAMP WITH TIME ZONE,
    implemented_at TIMESTAMP WITH TIME ZONE,
    
    -- 结果追踪
    outcome TEXT, -- 决策执行后的结果
    was_successful BOOLEAN,
    
    metadata JSONB DEFAULT '{}'::jsonb
);

CREATE INDEX idx_decisions_project_id ON decisions(project_id);
CREATE INDEX idx_decisions_task_id ON decisions(task_id);
CREATE INDEX idx_decisions_made_by_agent_id ON decisions(made_by_agent_id);
CREATE INDEX idx_decisions_decision_type ON decisions(decision_type);
CREATE INDEX idx_decisions_status ON decisions(status);
CREATE INDEX idx_decisions_proposed_at ON decisions(proposed_at DESC);
```

## 4. Redis 数据结构设计

### 4.1 Session Management

```
Key: session:{session_id}
Type: Hash
TTL: 24 hours
Fields:
  - user_id
  - created_at
  - last_activity
  - data (JSON)
```

### 4.2 Real-time Project State

```
Key: project:{project_id}:state
Type: Hash
TTL: 7 days
Fields:
  - status
  - current_task_id
  - active_agents (JSON array)
  - progress_percentage
```

### 4.3 Task Queue (Celery)

```
Queue: celery
Queue: celery:priority:high
Queue: celery:priority:low
```

### 4.4 Agent Communication Cache

```
Key: agent:{agent_id}:messages
Type: List
TTL: 1 hour
Value: JSON message objects
```

### 4.5 Rate Limiting

```
Key: ratelimit:{user_id}:{endpoint}
Type: String (counter)
TTL: 1 minute
```

## 6. 数据迁移策略

### 6.1 初始化脚本

```sql
-- 创建数据库
CREATE DATABASE ai_work_os;

-- 创建扩展
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
CREATE EXTENSION IF NOT EXISTS "pg_trgm";

-- 创建枚举类型
CREATE TYPE task_status AS ENUM ('pending', 'in_progress', 'blocked', 'review', 'approved', 'completed', 'failed');
CREATE TYPE message_type AS ENUM ('PROPOSAL', 'QUESTION', 'DISCUSSION', 'TASK_ASSIGNMENT', 'REPORT', 'DECISION', 'ERROR', 'INFO');
```

### 6.2 版本控制

使用Alembic进行数据库迁移管理:

```bash
alembic init migrations
alembic revision --autogenerate -m "Initial schema"
alembic upgrade head
```

## 7. 性能优化建议

### 7.1 索引策略

- 为所有外键创建索引
- 为常用查询字段创建复合索引
- 使用部分索引优化特定查询

### 7.2 分区策略

对大表进行分区:

```sql
-- 按时间分区 audit_logs
CREATE TABLE audit_logs_2026_03 PARTITION OF audit_logs
FOR VALUES FROM ('2026-03-01') TO ('2026-04-01');
```

### 7.3 查询优化

- 使用EXPLAIN ANALYZE分析慢查询
- 适当使用物化视图
- 使用连接池管理数据库连接
