## 2. 核心组件详解

### 2.1 User Interface Layer (前端层)

**技术栈**: Next.js 14 + React 18 + TypeScript + TailwindCSS + ReactFlow

**核心功能**:
- **Goal Input Interface**: 用户输入目标的智能表单,支持自然语言和结构化输入
- **Team Visualization**: 使用ReactFlow可视化展示AI团队结构和角色关系
- **Workflow Monitor**: 实时展示任务执行流程,支持节点状态更新
- **Agent Communication Panel**: 展示Agent之间的沟通记录,支持消息类型过滤
- **Result Dashboard**: 展示最终输出结果,支持责任追踪和版本对比

**关键特性**:
- 响应式设计,支持桌面和移动端
- WebSocket实时更新,无需刷新页面
- 支持暗黑模式
- 可导出工作流程图和沟通记录

---

### 2.2 API Gateway Layer (API网关层)

**技术栈**: FastAPI + Pydantic + WebSocket

**核心功能**:
- **Authentication & Authorization**: JWT认证,基于角色的权限控制
- **Request Routing**: 智能路由请求到对应的业务逻辑模块
- **WebSocket Manager**: 管理实时连接,推送任务状态更新
- **Rate Limiting**: 防止API滥用,支持用户级别限流

**API端点设计**:
```
POST   /api/v1/projects              # 创建新项目
GET    /api/v1/projects/{id}         # 获取项目详情
POST   /api/v1/projects/{id}/goals   # 提交目标
GET    /api/v1/projects/{id}/team    # 获取AI团队
GET    /api/v1/projects/{id}/workflow # 获取工作流
GET    /api/v1/projects/{id}/messages # 获取沟通记录
WS     /ws/projects/{id}             # WebSocket实时更新
```

### 2.3 Core Business Logic Layer (核心业务逻辑层)

#### 2.3.1 Goal Parser (目标解析器)

**职责**: 解析用户输入的自然语言目标,提取关键信息

**核心功能**:
- **Intent Detection**: 识别任务类型 (Research/Product Design/Development/Analysis/Startup)
- **Entity Extraction**: 提取关键实体 (技术栈、领域、时间限制、质量要求)
- **Constraint Analysis**: 分析约束条件 (预算、时间、资源)
- **Success Criteria**: 提取成功标准

**输出结构**:
```python
{
    "task_type": "software_development",
    "domain": "web_application",
    "requirements": {
        "tech_stack": ["Python", "FastAPI", "React"],
        "features": ["user_auth", "dashboard", "api"],
        "constraints": {
            "timeline": "2_weeks",
            "quality": "production_ready"
        }
    },
    "success_criteria": [
        "Fully functional API",
        "Responsive UI",
        "Test coverage > 80%"
    ]
}
```

---

#### 2.3.2 Team Generator (团队生成器)

**职责**: 根据任务类型自动生成最优AI团队

**核心功能**:
- **Role Matching**: 根据任务类型匹配所需角色
- **Team Composition**: 确定团队规模和结构
- **Capability Mapping**: 为每个Agent分配技能和工具
- **Authority Assignment**: 定义决策权限和审批流程

**团队模板库**:

#### 2.3.3 Workflow Generator (工作流生成器)

**职责**: 生成任务执行的工作流程图

**核心功能**:
- **Task Decomposition**: 将目标分解为子任务
- **Dependency Analysis**: 分析任务依赖关系
- **Parallel Optimization**: 识别可并行执行的任务
- **Critical Path**: 计算关键路径和预估时间

**输出**: LangGraph兼容的工作流图

#### 2.3.4 Accountability Engine (责任追踪引擎)

**职责**: 追踪每个决策和输出的责任人

**核心功能**:
- **Owner Tracking**: 记录每个任务的负责人
- **Decision Logging**: 记录所有关键决策及理由
- **Audit Trail**: 生成完整的审计日志
- **Blame Analysis**: 当出现问题时,追溯责任链

**数据结构**:
```python
{
    "task_id": "task_3",
```

### 2.4 Agent Orchestration Layer (Agent编排层)

**技术栈**: LangGraph + LangChain

#### 2.4.1 LangGraph Agent Orchestrator

**核心职责**: 管理Agent生命周期和任务执行

**关键组件**:

1. **Coordinator Agent** (协调者)
   - 接收用户目标
   - 分配任务给Planner
   - 监控整体进度
   - 处理异常和冲突

2. **Planner Agent** (规划者)
   - 制定详细执行计划
   - 分配任务给Specialist Agents
   - 优化执行顺序
   - 调整计划应对变化

3. **Specialist Agents** (专家Agent)
   - Research Agent: 信息收集和分析
   - Design Agent: 产品和架构设计
   - Code Agent: 代码编写和调试
   - QA Agent: 质量保证和测试
   - Writer Agent: 文档编写

4. **Communication Engine** (沟通引擎)
   - 消息路由
   - 消息类型管理 (PROPOSAL/QUESTION/DECISION/REPORT)
   - 沟通历史记录
   - 冲突解决机制

**LangGraph状态管理**:
```python
class AgentState(TypedDict):
    project_id: str
    goal: dict
    team: List[Agent]
    current_task: str
    task_queue: List[str]
    completed_tasks: List[str]
    messages: List[Message]
    decisions: List[Decision]
    outputs: List[Output]
    status: str  # "planning" | "executing" | "reviewing" | "completed"
```

### 2.5 LLM Integration Layer (LLM集成层)

**职责**: 统一管理多个LLM提供商

**核心功能**:
- **Multi-Provider Support**: 支持OpenAI、Anthropic、本地模型
- **Prompt Template Engine**: 管理和渲染Prompt模板
- **Response Parser**: 解析LLM响应,提取结构化数据
- **Token Management**: 追踪和优化Token使用
- **Fallback Strategy**: 主模型失败时自动切换备用模型
- **Cache Manager**: 缓存常见请求,降低成本

**配置示例**:
```python
LLM_CONFIG = {
    "coordinator": {
        "provider": "openai",
        "model": "gpt-4-turbo",
        "temperature": 0.3,
        "max_tokens": 2000
    },
    "specialist": {
        "provider": "anthropic",
        "model": "claude-3-sonnet",
        "temperature": 0.5,
        "max_tokens": 4000
    }
}
```

### 2.7 Data Persistence Layer (数据持久化层)

**技术栈**: PostgreSQL + Vector Database + Redis

**核心数据库**:

1. **PostgreSQL** - 结构化数据存储
   - 用户和项目数据
   - Agent和任务数据
   - 消息和决策记录

3. **Redis** - 缓存和队列

   - 任务队列 (Celery)

## 4. 关键技术决策

### 4.1 为什么选择LangGraph?

- **状态管理**: 原生支持复杂的Agent状态管理
- **工作流编排**: 可视化定义Agent协作流程
- **可观测性**: 内置执行追踪和调试工具
- **灵活性**: 支持动态修改工作流

### 4.2 为什么选择FastAPI?

- **高性能**: 基于Starlette和Pydantic,性能优异
- **异步支持**: 原生async/await,适合I/O密集型任务
- **自动文档**: 自动生成OpenAPI文档
- **类型安全**: Pydantic数据验证

### 4.3 为什么选择Next.js?

- **SSR/SSG**: 支持服务端渲染和静态生成
- **性能优化**: 自动代码分割和优化
- **开发体验**: 热重载、TypeScript支持
- **生态系统**: 丰富的React组件库

## 5. 扩展性设计

- **工具扩展**: 标准化工具接口,易于添加新工具

- **代码执行**: Docker容器隔离,资源限制
- **数据加密**: 敏感数据加密存储
- **审计日志**: 完整的操作审计追踪
